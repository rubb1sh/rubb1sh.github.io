发表于 2020-08-22   |   分类于 git

当同时开发两个branch，本来想在featureA分支，却在featureB分支上开发了，并完成了commit。这时，想把在featureB的这次commit转移到featureA上，可以使用git的cherry-pick。cherry-pick可以把在另外的branch的某次提交转移到当前的branch上。

为你要转移的commit哈希号（哈希号可以通过git log查看，不用完整的哈希号，取前几个就好了）
如果需要获取的commit是另一个branch的最后commit，可以使用一下命令，而无需commitHash。

转移多个分支

注意事项

  1. 使用cherry-pick转移分支后需要处理merge的冲突。
  2. 命令默认commit，同时使用转移的commit的提交信息。

其他选项

  • 如果想编辑提交记录，使用-e
  • 如果只更新工作区（git add），而不commit，使用-n，–no-commit
  • 冲突后，使用git cherry-pick –continue，完成后续分支转移
  • –abort放弃git cherry-pick操作（和git merge的操作类似）
