Reactor Pattern

发表于 2020-09-16   |   分类于 设计模式

在看Kafka客户端的网络实现使用了Reactor模式

背景

计算机网络中，一个服务往往会定义多个socket描述文件，用于获取不同的网络请求。然后一个过来的网络请求往往需要做多个处理，比如：可能会把一个网络请求打印到终端，同时又需要写入文件。使用Reactor模式，每个动作定义为一种handler（处理者），把handler和对应的socket文件描述文件绑定，当一个请求过来时，由一个中心控制器去找对应的handler进行处理。

Reactor结构

  • Initiation Dispatcher：请求分发器（上边提到的中心控制器），handler注册到其中，当请求来时（到Dispatcher)，Dispatcher找对应的handler处理（handler可以是多个）
  • Event Handler：抽象Handler
  • Concrete Event Handler：Handler的具体实现
